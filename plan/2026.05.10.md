# 2026.05.10 - 반복 탐지 및 VSIX 빌드 수정 계획

## 목표
- 반복 응답과 반복 편집 루프를 차단하는 로직을 점검하고 보강한다.
- `axios` 보안 취약점을 해결하기 위해 패키지를 최신 버전으로 업그레이드한다.
- 버전 번호를 갱신하고 VSIX 패키지를 새로 빌드한다.

## 수정 대상
- `src/repetitionWatchdog.ts`
- `src/chatPipeline.ts`
- `src/actionLoopGuard.ts`
- `src/fileStateGuard.ts`
- `src/chatPipelineHelpers.ts`
- `src/actionExecutor.ts`
- `tests/repetitionWatchdog-codeEditFix.test.mjs`
- `package.json`
- `package-lock.json`
- `scripts/package-vsix.mjs` (빌드 스크립트 검토)

## 작업 항목
1. 반복 탐지 로직 강화
   - 스트림 내에서 `action_tag`를 포함하는 반복 블록이 무조건 루프로 감지되지 않도록 예외 처리.
   - 동일한 `<edit_file>`/`<create_file>` 액션 블록이 여러 번 반복되는 경우 별도 탐지 로직 추가.
   - 코드 블록, 액션 블록, 마크다운 구조물에서 발생하는 false positive를 줄임.
   - 최근 블록 반복 감지 및 중요한 문장 반복 감지 로직을 재검토.

2. 액션/파일 반복 방지
   - `ActionLoopGuard`에 입력 본문 기반 fingerprint를 도입하여 동일 액션을 더 정확히 판별.
   - TTL을 기존 90초에서 120초로 늘려 짧은 재시도를 추가 방어.
   - `FileStateGuard`의 no-effect 편집 허용 횟수를 3회에서 5회로 늘려 정당한 반영 대기를 허용.
   - 동일 파일에 대한 반복 편집 시 상태 초기화 및 경고 메시지 처리 강화.

3. 이어받기 요청 품질 향상
   - `chatPipelineHelpers.buildContinuationSystemMessage()`에 `answer the user's request` 지시어를 추가하여 합리적인 후속 응답 유도.
   - 터미널 재시도/큐 관련 로직에서 반복 중단 후 재요청 방지 체크를 재검토.

4. 보안 및 종속성 업데이트
   - `axios`를 최신 안정 버전으로 업그레이드.
   - `npm install axios@latest --save` 실행 후 `package-lock.json`을 갱신.
   - `npm test`를 실행하여 수정 후 전체 테스트가 통과하는지 확인.

5. 버전 및 VSIX 배포
   - `package.json` 버전을 `3.3.35`로 갱신.
   - `npm run package:vsix -- --no-bump --notes "Bumped version and upgraded axios before VSIX build."`로 VSIX 패키지 생성.
   - 생성 산출물: `release/llem-3.3.35.vsix`

## 검증 항목
- `npm test` 전체 통과 여부
- `package-lock.json`에 `axios` 최신 버전 반영 여부
- `release/llem-3.3.35.vsix` 생성 여부
- 기존 반복 탐지/액션 루프 회귀 테스트 통과 여부

## 결과 예상
- 반복 출력/반복 편집 루프 방어가 강화된 상태.
- 보안 취약점이 있는 `axios` 버전을 제거.
- VSIX 패키지를 새 버전으로 안전하게 배포할 수 있는 상태.

## 비고
- `vsce` 패키저 버전이 `2.25.0`으로 오래된 상태였으며, 빌드 시 최신 버전 안내가 출력됨.
- 배포 전 추가로 `npm audit`와 `npm install -g @vscode/vsce` 검토를 권장.
