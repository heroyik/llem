# AI 무한 반복 감지 알고리즘 개선 계획

이전 구현이 무한 루프를 제대로 차단하지 못한 원인을 분석하고, 제안해주신 "중요한 메시지 블록을 기억했다가 반복을 감지"하는 방식에 맞춘 새로운 접근법을 제안합니다.

## 🚨 이전 알고리즘이 실패한 원인 (기존 방식의 한계)
이전 로직은 `ABCABCABC`처럼 **완벽하게 똑같은 문자열이 띄어쓰기 하나 틀리지 않고 꼬리를 물며 연속해서(연달아) 3번 등장해야만** 작동했습니다.
1. AI는 루프에 빠질 때 약간의 변형(띄어쓰기, 줄바꿈 추가 등)을 주거나, 중간에 다른 쓰레기 값(Garbage)을 넣고 다시 반복하는 경우가 많습니다 (`A... B... A... C... A`).
2. 기존 로직은 "연속된 패턴"만 찾았기 때문에 이런 비연속적인 반복이나 약간의 어긋남을 감지하지 못했습니다.

## 💡 새로운 해결 방안: "최근 윈도우 기억(Sliding Window) & 빈도수 체크" 방식

제안해주신 아이디어처럼 **가장 최근에 작성하고 있는 텍스트 블록(Memory Block)**을 떼어내서, 과거에 작성했던 내용들과 비교하는 방식이 훨씬 더 강력하고 유연합니다.

### 작동 원리 (제안)
1. **최근 블록 추출**: AI가 실시간으로 문장을 생성할 때, 가장 최근 생성된 **`N` 글자 (예: 100자)**를 하나의 블록으로 떼어냅니다.
2. **유효성 검사**: 이 100자의 블록이 단순히 괄호나 공백, 무의미한 반복(예: `</div></div>...` 또는 빈 줄 연속)이 아닌지 확인하기 위해, 영문/한글/숫자가 최소 30자 이상 포함되어 있는지 체크합니다.
3. **전체 기록 검색**: 이 유효한 100자 블록이 **지금까지 생성된 전체 답변 내에서 몇 번 등장했는지** 검색합니다.
4. **중단 조건**: 만약 완전히 똑같은 100자 이상의 텍스트 덩어리가 겹치지 않게 **3회 이상** 발견된다면, 이는 명백한 무한 루프이므로 즉시 생성을 중단(STOP)합니다.

### 이 방식의 장점
*   **비연속적 루프 감지**: 연달아 반복되지 않고 중간에 다른 내용이 섞여 있더라도, 핵심 문단이 반복되면 감지해냅니다.
*   **어긋남에 강함**: 루프의 시작점이나 끝점이 어긋나도, 100자라는 긴 고정 크기의 윈도우가 반복 패턴 안에 포함되기만 하면 무조건 감지됩니다.
*   **오작동 최소화**: 100자라는 긴 텍스트가 토씨 하나 틀리지 않고 3번이나 우연히 일치할 확률은 일반적인 코드나 텍스트 작성에서는 거의 0%에 가깝습니다.
